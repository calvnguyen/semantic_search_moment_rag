Section two: the baseline semantic search RAG — the conventional pipeline most teams ship first.
02 · Baseline Semantic Search RAG

The conventional pipeline.

Fixed chunks, one embedding lookup, stuff-and-answer.

Baseline query flow: transcript becomes fixed chunks, each chunk is embedded, a cosine similarity search returns the top-k closest chunks, and gpt-4o-mini answers from those. One retrieval signal, one pass.

6 · Generate the answer with gpt-4o-mini.

Single retrieval signal08 / 25
Baseline limitations: chunk boundaries are arbitrary because they fall on a token budget; a single explanation can be split across two chunks; and there is only one retrieval signal, so a single similarity lookup decides everything. These are exactly the gaps Moment RAG closes.
Baseline · Limitations09

Where it struggles.

Arbitrary structure

Chunk boundaries are set by a token budget, not by topic.

A single explanation can be split across chunks, so neither half is complete.

One signal

Only dense cosine retrieval — no keywords, no intent, no fusion.

One lookup decides everything; multi-part questions get under-answered.

The gap Moment RAG closes09 / 25
Section three: Moment RAG. The goal is to retrieve semantic moments instead of arbitrary chunks, and to move intelligence onto the query side.
03 · Moment RAG

Retrieve moments, not arbitrary chunks.

Seven stages: segment, enrich, hybrid retrieve, decompose, fuse, re-rank, ground.

Ingestion pipeline, done once. Transcript cues are grouped into semantic moments; each moment is enriched by GPT with a gist, keywords, and hypothetical questions; then everything is indexed three ways — moment text embeddings, question embeddings, and a BM25 keyword index.

Step 3, hybrid retrieval. Three retrieval systems run in parallel: dense text search compares the query embedding to moment text embeddings; dense question search compares it to the generated question embeddings; BM25 does keyword matching. Dense captures meaning, questions capture intent, BM25 captures exact keywords and acronyms.
Step 3 · Hybrid Retrieval15
Step 3

Three retrieval signals

Each captures something the others miss.

  • Dense text — query vs. moment-text embeddings → captures meaning
  • Dense questions — query vs. generated-question embeddings → captures intent
  • BM25 — keyword matching → captures exact terms & acronyms

decompose()16 / 25
Steps 5 and 6. RRF, reciprocal rank fusion, merges the rankings from dense, question, and BM25 retrieval — moments that rank highly across multiple methods get higher final scores. Then a cross-encoder, ms-marco-MiniLM-L-6-v2, re-scores the candidates against the original query. Embedding similarity is fast but approximate; the cross-encoder is slower but more accurate because it reads the query and moment together.

Key findings. Moment RAG improved recall, multi-part question coverage, overall retrieval quality, grounding, and explainability — all from how retrieval is structured rather than from a larger model.
Key Findings23
Key Findings

Moment RAG improved…

Same source, same model — better retrieval design.

  • Recall — finds the relevant moment more reliably
  • Multi-part coverage — decomposition answers every facet
  • Retrieval quality — hybrid + re-rank sharpen results
  • Grounding — timestamp citations into the video
  • Explainability — you can see why each moment was chosen
Outcome23 / 25
Limitations: a small transcript dataset, the cost of GPT enrichment, and added retrieval latency. Future improvements: replace the NumPy search with ChromaDB or Qdrant, use a larger evaluation dataset, add automated metrics like Recall@K, Precision@K, and MRR, and improve semantic segmentation.
